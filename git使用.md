# 🚀 团队 Git 协作极简指南 (小白版)

## 一、 核心概念：Git 到底是什么？

把 Git 想象成一个**“超级时光机”**和**“共享云盘”**：
1.  **时光机 (Version Control)**：它能记录代码的每一个改动。如果你写坏了，可以一键回到昨天。
2.  **共享云盘 (Remote Repo)**：大家把代码都传到云端（如 GitHub, Gitee 或 GitLab），实现多人同步。

---

## 二、 准备工作（仅需做一次）

1.  **安装 Git**：从 [git-scm.com](https://git-scm.com/) 下载并安装。
2.  **自报家门**：安装后，右键桌面选择 `Git Bash Here`，输入以下命令（让团队知道是谁改了代码）：
    ```bash
    git config --global user.name "你的中文名"
    git config --global user.email "你的邮箱"
    ```
3.  **下载项目**：从组长那里拿到项目链接（以 .git 结尾），在你想放代码的文件夹运行：
    ```bash
    git clone https://gitee.com/xxx/xxx.git
    ```

---

## 三、 每天工作的“黄金四步曲”

这是团队开发中最常用的流程，**请务必按顺序执行**：

### 第一步：同步远程代码（开工前）
在你写新代码之前，先要把队友昨天写的代码拉取到本地。
```bash
git pull
```

### 第二步：写代码
在你的 IDE（如 VS Code, PyCharm）里正常写代码、改 Bug。

### 第三步：提交到本地仓库（暂存成果）
代码写好了一个阶段，先保存在本地。
```bash
git add .                  # 把所有改动文件放进“待提交列表”
git commit -m "说明你改了什么"  # 备注信息一定要写清楚，严禁写 "111" 或 "update"
```

### 第四步：推送到云端（分享给队友）
这一步完成后，队友才能看到你的代码。
```bash
git push
```

---

## 四、 团队协作的三大铁律 ⚠️

### 1. 先 Pull，再 Push！
**严禁**直接 Push。如果云端有队友新传的代码，你直接 Push 会报错。
*   **正确做法**：先 `git pull`，确保本地代码是最新的，如果有冲突（见第五条）解决掉，然后再 `git push`。

### 2. 备注信息（Commit Message）要像人话
*   ❌ 错误：`git commit -m "fix"`
*   ✅ 正确：`git commit -m "修复了机械臂连接超时的Bug"`

### 3. 不要上传垃圾文件
有些文件不需要进 Git（比如 Python 的 `__pycache__`，或者临时生成的日志日志）。
*   **解决方法**：在项目根目录创建一个 `.gitignore` 文件，把不需要的文件名写进去。

## 五、 遇到“冲突（Conflict）”怎么办？（最怕的情况）

**现象**：当你 `git pull` 时，Git 提示 `Auto-merging failed; fix conflicts`。
**原因**：你和队友同时修改了**同一行代码**，Git 不知道听谁的。

**解决步骤**：
1.  打开报错的文件，你会看到类似这样的标记：
    ```text
    <<<<<<< HEAD
    你的代码（你写的）
    =======
    队友的代码（云端的）
    >>>>>>> 123456...
    ```
2.  **手动删掉**这些符号，把代码合并成正确的样子。
3.  保存文件。
4.  重新执行：`git add .` -> `git commit -m "解决冲突"` -> `git push`。

## 六、 常用命令小抄

| 命令 | 作用 | 什么时候用 |
| :--- | :--- | :--- |
| `git status` | 查看状态 | 忘了哪些文件改了，随时输入看看 |
| `git log` | 查看历史 | 想看看之前谁提交过代码 |
| `git clone` | 下载项目 | 第一次加入团队时用 |
| `git pull` | 拉取更新 | 每天早上开工第一件事 |
| `git add .` | 添加改动 | 写完一个功能后 |
| `git commit -m "..."` | 确认提交 | 写完一个功能后 |
| `git push` | 推送云端 | 确定代码没问题，要传给队友时 |

---

## 七、 给小白的特别建议

1.  **使用图形化工具**：如果你觉得命令行太可怕，可以安装 **GitHub Desktop** 或者在 **VS Code** 左侧自带的源代码管理面板操作。它们把命令变成了按钮，更直观。
2.  **小步快跑**：不要写了三天的代码才提交一次。建议每完成一个具体的小功能（比如：写好了一个函数）就 `commit` 一次。
3.  **别怕弄坏**：只要你执行了 `commit`，你的代码就进了数据库，几乎都能找回来。

**现在，大家可以先试着 `git clone` 项目，然后在里面新建一个 `名字.txt` 文件，尝试 `add`, `commit`, `push` 上去吧！**
